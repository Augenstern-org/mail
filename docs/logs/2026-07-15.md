# 2026-07-15 - SMTP 协议层

> 在 `mail::net` 网络层基础上，实现 SMTP 接收侧的协议解析、会话状态机与服务器接线。

## SMTP 协议层设计

### `mail::smtp::Command`

**命令行解析**——所有解析工作以游标推进方式在 `string_view` 上进行，不复制中间子串（仅在生成 domain/path 字段时构造 `std::string`）。

#### 设计要点

- **不依赖 locale**：ASCII 大小写转换自己实现，避免非 ASCII 环境的歧义；
- **参数校验**：对路径长度、参数数量等设施进行约束检验，失败返回专用 `ParseError` 码；
- **语义错误分离**：分类为 `Syntax`（格式错）、`PathTooLong`、`BadParam`（不支持的参数）、`None`（成功），便于映射到标准 SMTP 应答码。

支持命令：`EHLO`、`HELO`、`MAIL FROM`、`RCPT TO`、`DATA`、`RSET`、`QUIT`；`MAIL` 和 `RCPT` 支持 `SMTPUTF8` 参数。

### `mail::smtp::Session`

**会话状态机**——驱动完整的 SMTP 事务序列：`EHLO/HELO → MAIL → RCPT → DATA → QUIT`。

#### 核心职责

1. **状态转移**：
   - `Expect::Greeting` — 等待客户端 EHLO/HELO；
   - `Expect::MailFrom` — 等待 MAIL FROM 命令；
   - `Expect::RcptTo` — 等待至少一个 RCPT TO 命令；
   - `Expect::Data` — 等待 DATA 命令；
   - `Expect::Body` — 在 DATA 阶段，消费消息体至尾行"."；
   - `Expect::Done` — 完成事务，等待下一轮或 QUIT。
   
   转移遵循 RFC 5321 的严格时序。

2. **分帧安全**：
   - 与下层 `LineReader` 的分帧语义结合（严格 CRLF、裸 CR/LF 视为错误、超长行可恢复）；
   - `readCommandLine()` 返回前检查行长（剥除 CRLF 后）是否超过 `kMaxCommandLineOctets - 2`；
   - DATA 阶段用 `readBodyUntilDot()` 循环消费行，自行识别尾行"."，不依赖 LineReader（因为消息体允许嵌入任意字节）；
   - SMTP smuggling 风险由双层防护：LineReader 严格分帧 + Session 显式检查行长。

3. **回调接口**：
   - `MessageSink` — 接收完整消息（信封 + 消息体字符串视图）；
   - `RecipientVerifier` — 验证收件人地址是否被接受；
   - 二者可插拔，便于单元测试。

4. **超时与错误**：
   - 所有 I/O 返回 `IoStatus`，通过它判断对端关闭、阻塞、错误等情况；
   - 会话一旦入错误态，所有后续操作均返回预置的错误应答（不重复读取网络）。

### `mail::smtp::Reply`

**SMTP 应答格式**——支持单行应答（`2xx`、`5xx` 等）和多行应答（中间行以 `-` 续接、末行以空格分隔）。

```cpp
Reply::single(250, "OK");                    // 单行
Reply::multi({{ 250, "OK" }, { 250, "line2" }});  // 多行
```
