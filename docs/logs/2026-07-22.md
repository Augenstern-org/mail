# 2026-07-21 - 用户与认证（SMTP AUTH）

> AUTH 动词落地。

## 概览

libsodium 为本仓**第一个第三方依赖**。全树 WSL 构建通过，13/13 CTest 通过。

## 行长上限调整

RFC 4954 §4 明确 AUTH **续行**不受 SMTP 命令行长度限制。RFC 4616 要求 authzid/authcid/passwd 各支持 255 字节 → 最坏 767 字节原文 → base64 后 1024 → 含 CRLF 1026，超出现有 512 命令行上限与 LineReader 的 1001 分帧上限。

- 新增 `kMaxAuthLineOctets=2048`（含 CRLF）；AUTH 命令行本身仍受 512 约束；
- `kMaxWireLineOctets` = 各上下文上限的最大值，LineReader 据此分帧；
- **关键配合**：抬高分帧上限会使 1000..2046 字节的 DATA 行以 `Ok` 到达，若不补自查将静默废掉 RFC 5321 §4.5.3.1.6 的文本行上限。`collectData` 增设 Session 级自查兜回。LineReader 只负责"装下最长的合法行"，各上下文更紧的上限一律由 Session 逐一自查（命令行 / DATA 行 / AUTH 续行各一份）。

## 命令解析

- `Verb::Auth` + `parseAuth`：机制名按 RFC 4422 §3.1 校验（1–20 字符，`[A-Za-z0-9_-]`，折大写），区分"无初始响应"与"初始响应为 `=`"；
- **MAIL FROM `AUTH=` 参数**：RFC 4954 §5 MUST——宣告 AUTH 的服务器必须接受 `MAIL FROM` 上的 `AUTH=` 参数，即使客户端未认证。非中继服务器解析后**丢弃**（表现得如同 `AUTH=<>`）。`applyParam` 只按参数名白名单接受，`BOGUS=1` 等未知参数仍回 `BadParam`。

## `UserStoreVerifier` — 应用层适配器

header-only，`src/server/user_store_verifier.hpp`，namespace `mail::app`，依赖方向与 `MaildirSink` 相同。

- `verify` 先用与投递侧同一个 `sanitizedLocalPart` 归一并校验 local part，故凡回 `Accept` 的收件人 `MaildirSink` 必能投递，不会出现"RCPT 250、投递 451"的错配；
- 这是 M4 让 `RcptDecision::RejectPermanent`（550）**首次可达**的地方——在此之前 `AcceptAllVerifier` 使该分支恒不触发。

## `main.cpp` 改造

- `serveConnection` 新增 `std::shared_ptr<const UserStore>` 参数，按值传入工作线程（镜像 `root`），故 `main` 返回后仍运行的线程不悬空；
- 可选 `argv[3]` = 用户档路径。缺省则 AUTH 关闭 + `AcceptAllVerifier` 保留，完全向后兼容；提供则启动时 `UserStore::open`，失败硬退出；
- `initCrypto()` 在 accept 循环前一次性调用，库不可用即启动失败；
- 顶部不变量注释更新：工作线程现共享一份**只读的 UserStore**（经 `shared_ptr<const>` 按值传入、内容不可变、查询线程安全），除此之外仍不触碰共享状态。

## 构建集成

libsodium 经 **apt `libsodium-dev` + pkg-config** 取得（本机验证 1.0.18），非 vcpkg——`vcpkg.json` 保持为空。原因：`CMakePresets.json` 把生成器钉死 Ninja + `build/`，在 WSL 主构建线上不可用，vcpkg 工具链若要生效需手工传 `-DCMAKE_TOOLCHAIN_FILE`，得不偿失。

- `mailcommon` 链接 `PkgConfig::SODIUM` **PRIVATE**（公开头不暴露 sodium 类型）；
- 链接 `Threads::Threads` **PUBLIC**（`std::counting_semaphore` 在 Linux 需 pthread，六个 framework-free 测试目标靠 mailcommon 传播）。

## 当前边界

- **不支持**：TLS（属 M6——届时可解开"明文 AUTH 默认关闭"的张力，默认要求 TLS 后再允许 AUTH）、IMAP 读取、MIME 解析；
- **认证范围**：SASL PLAIN + LOGIN；不做 SASLprep / Unicode 规范化；无失败尝试计数 / 断连阈值；
- **配置**：用户档路径经 `argv[3]` 传入，无配置文件（`argv` 已占 port/root/users 三个位置参数，后续宜转配置文件）；
- **默认行为**：`requireAuthForMail` 默认 false——不提供用户档时服务器行为与今日完全一致，AUTH 为 opt-in；
- **测试覆盖**：13/13 CTest 通过 + 向后兼容冒烟（无 `argv[3]` 时正常启动到 accept 循环）；
- **下一步**：M5 读侧（cur/ 扫描、flag 解析）。
