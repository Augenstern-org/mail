# 2026-07-13 - SMTP 服务器

> 主要目的是练习网络编程与文件管理。

## 现有设施设计
`src/common/`中实现了底层设施，分两层：不依赖具体协议的**基础类型**（错误传递、fd 管理、上限常量），以及建立在其上的 `mail::net` **网络层**。下面分点叙述。

### 基础类型
后续的设施需要建立在这些底层基础之上。考虑到程序需要运行在Linux上，故采用Linux风格管理文件。

#### `IoStatus`
一切可能"部分成功"的 I/O 操作统一返回的状态码枚举类——`Ok` / `Closed` / `WouldBlock` / `Interrupted` / `LineTooLong` / `Timeout` / `Error`。约定是：**状态按值返回，产出的数据经出参传出**，二者分离。这样调用方一眼就能区分"对端正常关闭"（`Closed`）、"非阻塞暂时无数据"（`WouldBlock`）、"分帧违规或系统错误"（`Error`）等本质不同的情况，而不必去猜返回值的正负。附带一个 `toString`，供日志诊断，永不返回 `nullptr`。

#### `Result<T>`
"值-或-错误"类型，供**工厂函数**（如 `Listener::bind`）使用——它们要么产出一个 `T`，要么以一个 `IoStatus` 加捕获的 `errno` 失败。任一时刻"值"与"错误"恰有一个有意义，`operator bool` 指示是哪一个；失败分支的 `error()` 绝不会是 `IoStatus::Ok`。

- 与 `IoStatus` 的分工：**部分成功用 `IoStatus` + 出参，构造成败用 `Result<T>`**。

#### `FileDescriptor`
POSIX 文件描述符的仅移动 RAII 包装，纯头文件实现。绝不重复关闭（移动后原描述符更新为 `-1`）、析构在 `::close` 前后**保留 `errno`**（由于errno在线程内仅存在一份，因此需要提前保存）、负 fd 视为null且不能传给 `::close`。`Connection` 与 `Listener` 都通过 `FileDescriptor` 持有 `fd`，不必各自实现关闭逻辑。

#### `limits`
协议与安全上限，以字节（octet）计。来自 RFC 的值标注确切章节，项目自定的默认值（如 `LineReader` 的 8192 行长上限）则明确说明其为内存安全默认而非协议常量。
