# 2026-07-16 - SMTP 服务器接线

> 简单接入

## 服务器实现

### `src/server/main.cpp`

**一连接一线程的极简服务器**。

#### 架构
- 主线程 `accept` 循环，每个新连接以 `std::thread::detach` 派一个工作线程；
- 工作线程构造 `Session` 状态机，驱动完整事务序列直到 QUIT、对端关闭、超时或协议错误；
- `std::signal(SIGINT, ...)` 处理 Ctrl+C，对 listen socket 调 `::shutdown`，使 `accept` 返回 `EINVAL`，accept 循环干净退出。

#### 构件
- `AcceptAllVerifier` — 所有收件人都被接受（无需地址校验）；
- `DiscardSink` — 收到消息时打印一行信封摘要后丢弃（本进程仅做协议演练，不做真正投递）；
- `SessionConfig` — 可配置超时、行长上限等参数。

#### 并发特性（v1）
- detach 设计可接受的原因：工作线程启动后不触碰共享状态（每条连接通过 `ByteSink`/`ByteSource` 仅持有自己的 fd），故 SIGINT 时 main 返回不会导致崩溃。尚在连接中的工作线程在进程退出时被拆除；
- 多个工作线程可能并发写 stdout，v1 容忍行间交错（在实际部署中须改进）。

## 当前边界

- **测试覆盖**：单元测试覆盖 command parser、session 状态转移、reply 序列化；集成测试可用客户端连接服务器端验证完整事务；
- **下一步**：消息持久化、多线程的并发安全改进、性能优化。
