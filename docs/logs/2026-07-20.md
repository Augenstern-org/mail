## Maildir 存储

`mail::store` 命名空间，接口/实现分置 `include/mail/store/` + `src/store/`。分为两层：纯函数的唯一名生成层，以及带 I/O 的崩溃安全投递层。读侧（cur/ 扫描、flag 解析）留给 M5 IMAP。

### `maildir_name` — 唯一名生成（纯函数）

**零系统调用、零 I/O**。全部外部输入（时钟、微秒、pid、投递序号、主机名）通过 `MaildirNameParts` 显式注入，不在函数内取 `getpid()`、`gethostname()` 或 `gettimeofday`。这使得单元测试可以确定性地覆盖每段边界，无需 mock 任何东西。

#### 名字格式

沿用 DJB maildir(5) 与 Dovecot 的现代惯例：

```
<epoch>.M<usec>P<pid>Q<seq>.<escaped-host>
```

- 数字段一律十进制原样输出，不补零——`M0` 与 `M999999` 均合法；
- 主机名按 Maildir 约定转义：`/` → `\057`（字面反斜杠 + 三位八进制），`:` → `\072`；
- `makeMaildirUniqueName` 保证：结果不含 `:` `/`，不以 `.` 开头（hostname 为空时兜底 `"localhost"`）。

#### 设计意图

把"名字长什么样"和"从哪取时间/pid"拆成两层：`maildir_name` 只管拼字符串，`MaildirStore` 在 `deliverMessage` 时取实钟/实 pid/原子递增序号，注入纯函数层。测试可以直接构造 `MaildirNameParts{...}` 验证 us 边界、host 转义、仅 seq 不同即互异等场景，不做任何文件 I/O。

### `MaildirStore` — 崩溃安全的写侧存储

**仅移动**的 C++ 类，每个实例绑定一个 Maildir 根目录。工厂 `open(root)` 建 `root/{tmp,new,cur}`（0700，EEXIST 容忍），同时取主机名并转义一次，后续投递复用。

#### 投递序列（`deliverMessage`）

崩溃安全三步，参照 DJB 原始 maildir(5) 算法：

1. **独占创建** — `open(tmp/<unique>, O_CREAT|O_EXCL|O_WRONLY|O_CLOEXEC, 0600)`；
   - EEXIST（同微秒、同 pid、同序号碰撞）→ 换名重试，上限 16 次；
   - 16 次耗尽返回 `IoStatus::Error + errno`。
2. **全量写入** — `write` 循环直至所有字节落盘 → `fsync(file)` → `close` 检查；
   - data 按线上原样落盘（CRLF），不做任何转义或改写。
3. **原子发布** — `rename(tmp → new)` → `fsync(new/ 目录 fd)`；
   - 两次 fsync 保证文件数据与目录项都落盘后才对外可见。

**任一步失败**：`unlink` 自己的 tmp 文件后返回 `IoStatus::Error + errno`（保留首个失败点 errno）。不会留半截垃圾在 tmp/ 里。

#### 关于 rename 的原子性

`rename(2)` 在同文件系统内是原子的。`MaildirStore` 不跨设备——tmp/ 和 new/ 永远在同一 root 下——因此投递要么完整出现在 new/，要么完全不存在。不会出现"文件在 new/ 但内容未写完"的中间态。

### `MaildirSink` — SMTP → Maildir 适配器

header-only，位于 `src/server/maildir_sink.hpp`，namespace `mail::app`。实现 `smtp::MessageSink`，把每封收到的信逐收件人投一份独立拷贝。

#### 依赖方向

适配器在应用层（`src/server`），反向依赖 storage 层的 `MaildirStore`。storage 层绝不 include `MessageSink`，以免协议语义泄漏进存储层。这是 M2 就定下的约束，M3 严格执行。

#### 投递行为

- **每收件人一份**：遍历 envelope 的 recipients，各投一份到 `<root>/<sanitized-local-part>/new/`；
- **local-part 白名单**：`sanitizedLocalPart` 校验字符集 `[A-Za-z0-9._+-]`、非空、不以 `.` 开头、不含 `..`——任一不满足返回空串，触发整体 TransientFail；
- **邮箱按需 open**：`MaildirStore::open(root + "/" + sanitized)`，首次投递到某邮箱时自动建目录。

#### 错误语义

**绝不返回 PermanentFail**。存储侧的一切失败——磁盘满、权限不足、非法收件人 local part——一律映射为 TransientFail（451）。理由：收件人的最终合法性应在 RCPT 阶段由 verifier 把关，到 `MessageSink::deliver` 这一步已是暂时性的落地问题，客户端可稍后重试。

#### 重试重复语义

这是刻意取舍：部分收件人成功、部分失败时，已成功落入 new/ 的拷贝不回收。客户端收到 451 后重试会对先前成功的收件人再投一份，产生重复投递。回收已落盘副本本身也可能失败，且跨收件人原子性并非本层职责。

### `main.cpp` 改造

`DiscardSink`（打印一行摘要就扔掉）整体移除。`serveConnection` 改签名为 `(Connection, string root)`——每个工作线程按值持有 root 拷贝，构造 `MaildirSink` 接入 `Session`。`main` 从 `argv[2]` 取 Maildir 根目录，默认 `"./mailroot"`。

至此 server 从"协议演练"变为**真正落盘的 SMTP 服务**：`telnet 127.0.0.1 2525` 完成 EHLO → MAIL → RCPT → DATA 后，消息会出现在 `<root>/<local-part>/new/` 中，CRLF 原样、`tmp/` 为空。

| `maildir_sink_test` | 临时目录集成 | `sanitizedLocalPart` 纯函数全集；单/多收件人成功投递（各一份独立拷贝）；非法收件人整体 451；部分失败不回收已落盘副本；root 不可写 → 451；空消息；断言绝不返回 PermanentFail |

## 当前边界

- **不支持**：TLS、认证、IMAP 读取、MIME 解析；
- **存储侧不支持**：读（cur/ 扫描、flag 解析）——属于 M5 IMAP；
- **不支持**：跨收件人原子回滚——部分失败时已落盘副本保留（见上文重试重复语义）；
- **测试覆盖**：8/8 CTest 通过 + 活体冒烟（250 后 new/ 恰一文件、CRLF 原样、tmp/ 空）；
- **下一步**：M4 用户与认证——`auth::UserStore`/`Password`/`Sasl`，`RecipientVerifier` 从 AcceptAll 换为真实校验。
